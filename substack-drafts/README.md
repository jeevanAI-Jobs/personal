# Substack Drafts — Jeevan AI Founder Newsletter

Newsletter issues staged for https://jeevanaifounder.substack.com, ready to copy-paste.

These are written in a personal founder voice (first person, email tone), not verbatim blog copies. Each one is repurposed from a published jeevanai.co.in blog post so the newsletter and blog reinforce each other without duplicating word-for-word.

Each .txt file is laid out as:
- **Line 1:** suggested email subject line / post title
- **Line 2 (after blank):** subtitle / preview text
- **Body:** the issue content (Substack understands bold with **asterisks**, lists, and headings)
- **Bottom:** the canonical blog URL and posting notes (everything below the dashed line stays out of the post)

## Posting steps (2-3 min per issue)
1. Open the .txt file on GitHub and click **Raw** to see clean plain text
2. Copy everything ABOVE the dashed line (`- - -`)
3. Open substack.com → your publication → **New post**
4. Paste the subject line into the title, the subtitle into the subtitle field, and the body below
5. Set the canonical URL: post **Settings → SEO → Canonical URL** → paste the original blog link (avoids duplicate-content dilution against the blog)
6. Add tags if you use them, then publish / send

## Why canonical matters
The blog post is the SEO and AI-citation asset. The Substack issue is for distribution and audience. Pointing the canonical at the blog tells search and AI engines the blog is the source of record, so the newsletter amplifies reach without competing with the original.
